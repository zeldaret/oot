958  * Data Structures
959  *
960  * NOTE:
961  * The DMA transfer hardware requires 64-bit aligned, 64-bit multiple-
962  * sized transfers. This important hardware optimization is unfortunately
963  * reflected in the programming interface, with some structures
964  * padded and alignment enforced.
965  *
966  * Since structures are aligned to the boundary of the "worst-case"
967  * element, we can't depend on the C compiler to align things
968  * properly.
969  *
970  * 64-bit structure alignment is enforced by wrapping structures with
971  * unions that contain a dummy "long long int". Why this works is
972  * explained in the ANSI C Spec, or on page 186 of the second edition
973  * of K&R, "The C Programming Language".
974  *
975  * The price we pay for this is a little awkwardness referencing the
976  * structures through the union. There is no memory penalty, since
977  * all the structures are at least 64-bits the dummy alignment field
978  * does not increase the size of the union.
979  *
980  * Static initialization of these union structures works because
981  * the ANSI C spec states that static initialization for unions
982  * works by using the first union element. We put the dummy alignment
983  * field last for this reason.
984  *
985  * (it's possible a newer 64-bit compiler from MIPS might make this
986  * easier with a flag, but we can't wait for it...)
